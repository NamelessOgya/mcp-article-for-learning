# Python Docker 開発環境

このプロジェクトは、Pythonアプリケーションの開発向けに`Dockerfile`と操作用スクリプトを提供します。

## ファイル構成
- `Dockerfile`: Pythonの実行環境と必要なパッケージ (`requirements.txt`) を定義した設定ファイルです。
- `requirements.txt`: インストールするPythonパッケージのリストです。
- `scripts/`: コンテナを簡単に操作するための各種シェルスクリプト群です。

## スクリプトの利用方法

コンテナの操作は `scripts` フォルダ内のファイルを利用することで簡単に行えます。LinuxやmacOSなどのターミナルから以下の手順で実行してください。

### 0. 実行権限の付与（初回のみ）
スクリプトに実行権限がついていない場合は、初回に以下のコマンドで付与してください。
```bash
chmod +x scripts/*.sh
```

### 1. コンテナの起動 (`scripts/start.sh`)
以下のスクリプトを実行すると、Dockerイメージのビルドと、バックグラウンドでのコンテナ起動が行われます。ホスト（手元の環境）にあるファイルはコンテナ内部の `/app` に同期されるため、編集内容はすぐに反映されます。

```bash
./scripts/start.sh
```

### 2. コンテナに入る (`scripts/enter.sh`)
コンテナが起動している状態で以下のスクリプトを実行すると、コンテナ内のシェル (`bash`) を立ち上げて操作可能になります。ここで `python script.py` などのコマンドを実行できます。

```bash
./scripts/enter.sh
```
※終了して元のターミナルに戻るには `exit` と入力します（コンテナ自体は動き続けます）。

### 3. コンテナの停止 (`scripts/stop.sh`)
作業が完了したら、以下のスクリプトでコンテナを停止します。停止と同時に不要なコンテナは自動で削除されます。

```bash
./scripts/stop.sh
```
